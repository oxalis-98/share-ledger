# <img src="ledger_icon.svg" width="36" height="36" valign="middle" alt="ロゴ"> 清算帳簿 (Co-Ledger App)

複数人（同居、カップル、シェアハウス、グループ旅行など）で日々の支出を記録し、誰が誰にいくら送金・精算すべきかを自動計算・管理できる、モダンで高機能な共同家計簿アプリケーションです。
Google Apps Script (GAS) をバックエンドとし、完全にサーバーレスで動作します。

---

## ✨ 主一な特徴

1. **超高速な表示体験 (Stale-While-Revalidate キャッシュ & プリフェッチ)**
   - セッションキャッシュによる読み込み時間ゼロの「SWRパターン」を採用。画面遷移や月・年切り替え時は一瞬で仮描画し、バックグラウンドでサーバーの最新タイムスタンプと同期します。
   - 表示月の前後月データを事前にバックグラウンドでプリフェッチしておくことで、ページめくりもストレスフリーです。
2. **AIレシート自動解析 (Gemini API & Googleドライブ連携)**
   - ローカルまたはGoogleドライブ上の画像・PDFレシート（複数ページ・複数枚の一括解析対応）を選択し、Gemini 1.5 Pro / Flash が店名、日付、カテゴリ、金額などを瞬時に自動解析します。
   - 解析結果を一括でプレビュー・編集してスプレッドシートへワンクリック登録できます。
3. **柔軟な精算・負担ルール**
   - 支出ごとに「割り勘（全員）」「立て替え（自分以外全員）」「個別（自分のみ）」などの精算タイプを選択可能。
   - 負担対象メンバーを個別にマルチセレクトして、均等割りを細かく調整することもできます。
4. **直感的な精算アコーディオンUI**
   - 月ごとの精算ログ（支払った・支払い済みなど）をすっきり管理するアコーディオンUIを搭載。
   - 精算パターンやステータスは、スプレッドシート上のマスタデータを書き換えるだけでリアルタイムに反映・動的同期されます。
5. **洗練されたモダンなダークモードデザイン & 完全レスポンシブ**
   - ダークテーマを基調とした目に優しいガラスモルフィズムUI。
   - モバイル環境に最適化されており、スマホの画面でもデバッグコンソールや操作ボタン、カレンダーUIが綺麗にフィットします。
6. **アプリ内デバッグコンソール**
   - URLパラメータに `?debug=true` を付与してアクセスすると、画面最下部に「デバッグコンソール」が出現。
   - データベース（キャッシュ）、API通信、UI表示、プリフェッチ処理などのバックグラウンドイベントがカラーバッジ付きでリアルタイムロギングされ、ワンクリックでコピーできます。

## 🛠 フォルダ構成

GitHubで公開する際は、以下のファイルを同一プロジェクト（またはclasp環境）内に配置します。

* `Code.js` : バックエンド処理（スプレッドシートの操作、Gemini API呼び出し、Googleドライブ連携など）
* `index.html` : フロントエンドのメインHTML（全体のレイアウトとモーダル定義）
* `js.html` : クライアントサイドのJavaScriptロジック（SWR、プリフェッチ、通信制御、ログ収集など）
* `css.html` : スタイリング定義（洗練されたダークテーマ、ガラスモルフィズム、ローディング・同期アニメーションなど）
* `ledger_icon.svg` : アプリケーションのロゴマーク（パス化された高解像度SVG）
* `appsscript.json` : GASのプロジェクト設定ファイル

## 🚀 導入・デプロイ方法

### 1. スプレッドシートの用意
1. Googleスプレッドシートを新規作成します。
2. 作成したスプレッドシートのURLから、**スプレッドシートID**（`/d/` と `/edit` の間にある長い英数字の文字列）をコピーします。
3. スプレッドシートの「共有」設定で、家計簿を利用するメンバー全員が編集できる権限を付与します。

### 2. GASプロジェクトの作成とコードの配置
1. スプレッドシートのメニューから「拡張機能」 ➜ 「Apps Script」を開きます。
2. 元からある `コード.gs` を `Code.gs` にリネーム（または削除して新規作成）し、本リポジトリの `Code.js` の内容を貼り付けます。
3. 同様に、ファイル追加ボタン（`+` マーク）から「HTML」を選択し、以下の3つのファイルを新規作成してそれぞれのコードを貼り付けます：
   - `index` (※自動的に `.html` になります) ➜ `index.html` の中身
   - `js` ➜ `js.html` の中身
   - `css` ➜ `css.html` の中身

### 3. スプレッドシートIDとAPIキーの設定
安全にアプリを動作させるため、APIキーやスプレッドシートIDは「スクリプトプロパティ」に保存して使用します。

1. GASのエディタ画面左側にある歯車マーク（「プロジェクトの設定」）をクリックします。
2. 一番下の「スクリプトプロパティを追加」をクリックし、以下のプロパティを登録します。

| プロパティ名 | 設定値 | 説明 |
| :--- | :--- | :--- |
| `SPREADSHEET_ID` | （手順1でコピーしたID） | アプリのデータを格納するスプレッドシートのID |
| `GEMINI_API_KEY` | （Google AI Studioから取得したキー） | レシート解析機能を使用する場合に必要（任意） |

> **Note**: `GEMINI_API_KEY` は、アプリの起動後に右上にある「設定（歯車マーク）」モーダルからブラウザ上で直接設定・登録することも可能です。

### 4. ウェブアプリとしてデプロイ
1. GASエディタ右上の「デプロイ」 ➜ 「新しいデプロイ」をクリックします。
2. 種類の選択（歯車マーク）で「ウェブアプリ」を選択します。
3. 設定を以下のように設定します：
   - **次のユーザーとして実行**: `ウェブ アプリケーションにアクセスしているユーザー` (User accessing the web app)
   - **アクセスできるユーザー**: `Google アカウントを持つ全員` (Anyone with Google account)
     > **Important**: この設定にすることで、**スプレッドシート本体の共有権限（編集権限）を持つメンバーのみが家計簿アプリにデータを読み書きできる**ようになります。これにより、URLを知られても第三者にデータを覗き見されたり改ざんされたりしない、安全なアクセス制御（GSSの権限連携）が有効になります。
4. 「デプロイ」ボタンを押し、表示される「ウェブアプリのURL」をコピーしてブラウザで開きます。

## 📂 スプレッドシートのデータ構造（自動初期化）

アプリを初めてデプロイして起動すると、指定したスプレッドシート内に以下の3つのシートが自動的に作成・初期化されます。手動でシートを作る必要はありません。

### 1. `transactions` シート (取引明細)
日々の支出データが格納されます。
* カラム： `id`, `date`, `title`, `category`, `amount`, `paid_by`, `split_type`, `beneficiaries`, `memo`, `created_at`

### 2. `masters` シート (マスタ設定)
アプリ内のカテゴリや利用メンバー、精算の組み合わせなどを一元管理します。ここのセルの値を編集することで、アプリ側のドロップダウンやボタンの選択肢が動的に変化します。
* カラム：
  1. `categories`: 「食費」「日用品」「光熱費」などの支出カテゴリ
  2. `users`: アプリを利用するメンバー名（初期状態では「ユーザーA」「ユーザーB」「ゲスト」が登録されます）
  3. `split_types`: 「割り勘」「立て替え」「個別」などの分割タイプ
  4. `payment_methods`: 精算時に選べる支払い手段（「振込」「現金」など）
  5. `settlement_statuses`: 精算のステータス（「未払い」「支払い済み」など）
  6. `payment_patterns`: 「AからBへ」といった自動生成される清算送金の組み合わせパターン

### 3. `settlements` シート (精算ログ)
月末の送金や精算の記録が保存されます。
* カラム： `id`, `month`, `status`, `payment_date`, `payment_pattern`, `amount`, `payment_method`, `memo`, `updated_at`

## 💻 ローカル開発環境の構築 (claspを使用する場合)

もしローカルのコードエディタ（VS Codeなど）で開発を行い、GitHubでソース管理しつつGASへデプロイしたい場合は、Google公式のコマンドラインツール `clasp` の使用を推奨します。

1. **CLIのインストール**
   ```bash
   npm install -g @google/clasp
   ```
2. **ログイン認証**
   ```bash
   clasp login
   ```
3. **プロジェクトのクローン/作成**
   スプレッドシートと紐づいた既存のGASプロジェクトがあるディレクトリで、以下を実行して設定ファイルを関連付けます。
   ```bash
   clasp clone "YOUR_GAS_SCRIPT_ID"
   ```
   > **Warning**: GitHubに公開する際は、`.clasp.json` や `.clasprc.json` などの個人用の設定ファイルがアップロードされないよう、`.gitignore` を適切に設定してください（本リポジトリにはあらかじめ設定済みの `.gitignore` が含まれています）。
